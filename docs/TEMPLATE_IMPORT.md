# 旧文件生成模板与全文编辑说明

## 什么是“从旧文件生成模板”

“从旧文件生成模板”用于把已有产品说明书、操作手册、历史模板或截图中的写作结构抽象为可复用模板。它不是简单把旧文档原文塞进提示词，而是通过模型提取标题层级、段落写法、字段规则、语气规则、必须包含项和禁止事项。

## 支持的文件

- `.txt`
- `.md`
- `.docx`
- `.pdf`
- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

所有文件都在浏览器中解析或 OCR，不上传到第三方文件存储服务。图片使用 OCR 提取文字；可复制文本 PDF 使用 pdf.js 提取文本；Word 文档使用 mammoth 提取纯文本。

## 如何解析模板

1. 进入 Templates 页面。
2. 在“从旧文件生成模板”区域上传旧说明书或截图。
3. 等待文件解析或 OCR 完成。
4. 填写“我希望模仿这份文档的哪些特点”。
5. 点击“解析生成模板”。
6. 检查生成出的模板名称、用途说明、标题层级、字段规则和 Prompt Instruction。
7. 点击“保存为自定义模板”。

## 模板会提取哪些内容

- 标题层级规则
- 段落写作模式
- 功能入口、操作步骤、字段说明等字段规则
- 语气规则
- 必须保留的内容
- 禁止复制或编造的内容
- 可直接进入生成 prompt 的模板约束

## 模板和参考材料的区别

参考材料上传用于本次生成时提供上下文，适合补充资料、截图 OCR 和说明片段。

旧文件生成模板用于抽象旧文档的结构和写法，保存后可以反复复用，适合让后续文档稳定遵循同一套标题层级、段落组织和字段说明方式。

## 如何使用自定义模板生成新说明书

1. 在 Templates 页面保存自定义模板。
2. 在自定义模板卡片中点击“启用模板”。
3. 进入 Workspace 后选择 Demo 项目和功能模块。
4. 填写关键词、参考写法和特别要求。
5. 点击生成提示词、调用 API 生成正文或生成 Mock 文档。
6. 生成 prompt 会优先包含所选自定义模板的结构和写作规则。

## 如何进入全文编辑页面

在 Workspace 的“文档预览”Tab 中生成或粘贴正文后，点击“打开全文编辑”。系统会把当前正文、模板信息、基础信息和参考材料摘要保存为 active document，并打开 `/editor`。

## 如何用 AI 修改助手优化文本

全文编辑页面右侧提供 AI 修改助手，支持修改全文、修改选中文本、追加内容和润色当前段落。可以使用快捷指令，例如：

- 改成更正式的产品说明书语气
- 补充操作步骤
- 补充功能价值
- 压缩选中文本
- 检查是否遗漏功能入口、关键字段、权限边界

AI 返回结果后不会自动覆盖正文，需要用户手动选择应用到原文、替换选中文本、追加到文末或放弃。

## 当前限制

- OCR 质量依赖图片清晰度。
- Word 解析只提取纯文本，不保留复杂样式。
- PDF 扫描版可能无法直接提取文本，建议转为图片后上传 OCR。
- localStorage 有容量限制，超大模板或大量 OCR 文本可能保存失败。
- 模板解析和 AI 修改需要 `.env.local` 中配置可用模型服务；未配置时仍可使用本地 Mock 文档和提示词模式。
